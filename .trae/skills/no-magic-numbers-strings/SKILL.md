# 禁止魔法数字/字符串

## 触发条件
当代码中出现以下情况时强制应用本规范：
- 在逻辑判断、函数调用、localStorage 读写、HTML 属性赋值等位置
- 直接使用字面量数字或字面量字符串，而不是具名常量

## 规范内容
所有在多处出现、或具有业务含义的数字和字符串，必须在文件顶部的 constants 模块中集中定义为 const 常量，然后在所有使用处引用该常量名。

### 必须常量化的情况：
1. **枚举值**：如状态码 0/1/2/3 → 定义为对象常量如 `const STAGE = { NEGOTIATING: 0, ... }`
2. **localStorage 键名**：如 'ai_provider' → 统一收录进 STORAGE_KEY 对象
3. **业务阈值**：如 5 * 1024 * 1024 → 定义为 `const MAX_STORAGE_BYTES`
4. **只要字面量出现超过 1 次，或含有业务语义，就必须常量化**

### 例外情况：
无业务含义的通用值不需要常量化，如：
- 0（数组初始索引）
- ''（空字符串默认值）
- true/false

## 检查方法
1. 搜索代码中的字面量数字和字符串
2. 检查是否在多处使用或具有业务含义
3. 验证是否已定义为常量
4. 确保所有使用处都引用常量名

## 修复方法
1. 在文件顶部的 constants 模块中添加相应的常量定义
2. 替换所有使用处的字面量为常量引用
3. 确保常量命名清晰，能准确表达其业务含义
