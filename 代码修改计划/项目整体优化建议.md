> 创建时间：2025-01-XX

---

## 一、项目架构概述

这是一个基于浏览器的项目管理应用，采用模块化设计，包含以下核心模块：

| 模块 | 路径 | 职责 |
|------|------|------|
| UI层 | project-manager.html | 看板视图、表单模态框、侧边栏面板 |
| 数据层 | js/core/db.js | IndexedDB + localStorage 双存储策略 |
| 业务层 | js/core/project.js, js/features/* | 项目管理、回款管理、待办分析 |
| 文件系统 | js/file/system.js | 本地文件夹三层存储映射系统 |
| AI层 | js/ai/engine.js | 多Provider支持、任务槽位系统 |

### 技术特点

- **动态导入**：main.js 使用 `import()` 实现代码拆分和懒加载
- **双存储策略**：IndexedDB 存储结构化数据，localStorage 用于缓存和配置
- **AI 多 Provider**：支持 Claude、OpenAI、Gemini、自定义 Provider
- **File System Access API**：浏览器原生文件系统访问

## 二、修改意见

### 1. 架构与代码组织

**问题**：全局变量过多，模块间耦合度较高

**建议**：
- 引入 ES Module 的依赖注入模式，减少 `window` 对象的滥用
- 建立统一的状态管理机制（如使用简单的 store 模式）

**具体修改**：

创建 `js/core/store.js`：

```javascript
export const store = {
  projects: [],
  recycleBin: [],
  statsFilter: 'thisYear',
  currentEditProjectId: null,
  editingId: null,
  db: null,
  fsRootHandle: null,
  projectDirMap: {},
};

export function setState(key, value) {
  store[key] = value;
}

export function getState(key) {
  return store[key];
}
```

在各模块中替换 `window.projects` 为 `import { store } from './store.js'`。

### 2. 数据库模块优化

**问题**：`db.js` 中 `saveProject` 方法过于冗长，重复的 cursor 遍历逻辑

**建议**：提取公共的批量删除逻辑

**具体修改**（js/core/db.js）：

新增辅助方法：

```javascript
async deleteByIndex(store, indexName, value) {
  const index = store.index(indexName);
  const cursorRequest = index.openCursor(value);
  await new Promise((resolve) => {
    cursorRequest.onsuccess = (event) => {
      const cursor = event.target.result;
      if (cursor) {
        store.delete(cursor.value.id);
        cursor.continue();
      } else {
        resolve();
      }
    };
  });
}
```

简化 `saveProject` 中的删除逻辑：

```javascript
await this.deleteByIndex(stores.paymentNodes, 'projectId', project.id);
await this.deleteByIndex(stores.collectTasks, 'projectId', project.id);
await this.deleteByIndex(stores.todos, 'projectId', project.id);
await this.deleteByIndex(stores.logs, 'projectId', project.id);
```

### 4. 渲染性能优化

**问题**：`render.js` 中的 `cardHTML` 函数缓存策略不完善，缺少缓存失效机制

**建议**：添加缓存清理时机和缓存大小限制

**具体修改**（js/core/render.js）：

```javascript
const renderCache = new Map();
const MAX_CACHE_SIZE = 500;

function clearRenderCache(projectId = null) {
  if (projectId) {
    renderCache.forEach((_, key) => {
      if (key.startsWith(projectId)) {
        renderCache.delete(key);
      }
    });
  } else {
    renderCache.clear();
  }
}

function cardHTML(p, colKey) {
  const cacheKey = `${p.id}_${colKey}_${p.updatedAt || ''}_${p.version || 0}`;

  if (renderCache.size > MAX_CACHE_SIZE) {
    const keys = Array.from(renderCache.keys()).slice(0, Math.floor(MAX_CACHE_SIZE / 2));
    keys.forEach(key => renderCache.delete(key));
  }

  if (renderCache.has(cacheKey)) return renderCache.get(cacheKey);

  // ...原有渲染逻辑

  renderCache.set(cacheKey, html);
  return html;
}
```

### 5. AI 模块改进

**问题**：`engine.js` 中的 Provider 配置较为分散，缺少统一的错误处理

**建议**：添加统一的错误处理中间件和超时控制

**具体修改**（js/ai/engine.js）：

```javascript
async function aiCallWrapper(callFn, options) {
  const { task, maxRetries = 3, timeout = 60000 } = options;

  for (let retry = 0; retry < maxRetries; retry++) {
    try {
      const controller = new AbortController();
      const timeoutId = setTimeout(() => controller.abort(), timeout);

      const result = await callFn({ signal: controller.signal });
      clearTimeout(timeoutId);
      return result;
    } catch (error) {
      if (error.name === 'AbortError') {
        throw new Error(`AI 请求超时（${timeout/1000}秒）`);
      }
      if (retry === maxRetries - 1) {
        throw error;
      }
      await new Promise(resolve => setTimeout(resolve, Math.pow(2, retry) * 1000));
    }
  }
}
```

### 6. 文件系统模块改进

**问题**：`system.js` 中的 `getProjectDirById` 函数逻辑复杂，多层嵌套

**建议**：重构为更清晰的步骤化逻辑

**具体修改**（js/file/system.js）：

```javascript
async function getProjectDirById(projectId) {
  if (window.projectDirMap[projectId]) {
    return window.projectDirMap[projectId];
  }

  if (!window.fsRootHandle) {
    return null;
  }

  const dir = await tryRecoverFromStorage(projectId);
  if (dir) return dir;

  return await scanForProjectDir(projectId);
}

async function tryRecoverFromStorage(projectId) {
  const sources = [getDirMapFromStorage, loadIndexFromRoot];

  for (const getMap of sources) {
    const map = await getMap();
    if (map[projectId]) {
      try {
        const dir = await window.fsRootHandle.getDirectoryHandle(map[projectId]);
        window.projectDirMap[projectId] = dir;
        return dir;
      } catch (e) {}
    }
  }
  return null;
}
```

### 7. 安全改进

**问题**：敏感配置（API Key）存储在 localStorage 中，未加密

**建议**：使用 `sessionStorage` 存储敏感信息，或采用加密存储

**具体修改**（js/core/project.js）：

```javascript
const SECURE_KEY_PREFIX = 'pm_secure_';

function saveSecureConfig(key, value) {
  try {
    sessionStorage.setItem(SECURE_KEY_PREFIX + key, value);
  } catch (e) {
    localStorage.setItem(key, value);
  }
}

function getSecureConfig(key) {
  try {
    return sessionStorage.getItem(SECURE_KEY_PREFIX + key) || localStorage.getItem(key);
  } catch (e) {
    return localStorage.getItem(key);
  }
}
```

### 8. 错误边界处理

**问题**：缺少全局错误捕获机制

**建议**：添加统一的错误处理和用户友好的错误提示

**具体修改**（main.js）：

```javascript
window.addEventListener('error', (event) => {
  if (DEBUG) console.error('全局错误:', event.error);
  showToast(`❌ 系统错误：${event.error.message || '未知错误'}`);
});

window.addEventListener('unhandledrejection', (event) => {
  if (DEBUG) console.error('未处理的 Promise 拒绝:', event.reason);
  showToast(`❌ 操作失败：${event.reason?.message || '未知错误'}`);
});
```
