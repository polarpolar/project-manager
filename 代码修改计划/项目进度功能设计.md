# 项目进度功能设计文档

## 一、功能概述

从语雀表格导入项目时，自动识别并同步每个项目的月度进度记录。进度数据单向从语雀同步到本地，不支持手动编辑。

1、在导入数据的时候，我的yuque表格中还有更新进度的文字描述。表格中逻辑是这样的：\
（1）每一行是一个项目，前几列是项目的各项信息，后面的列是项目最近的进度清理，定期更新；\
（2）项目进度的逻辑是，每个月会新增一个列，用于保存当月的新进度；\
（3）在同一个月内，进度会直接覆盖在当前的表格中；\
（4）进入下一个月的时候，会新增一个列（名称例如2026年4月），然后把最新的进度更新在里面；\
（5）这样的话， 就形成了每个月“1条”进度的样子（当然，每个月的进度在当月内是不断更新的）；\
（6）我们默认，以更新的时间为准，例如，有些工作是3月底做的，但如果是4月初更新的表格，这些进度就会更新在4月份的格子里面。

2、我希望实现的功能大致是这样的：\
（1）导入yuque的时候，检查项目的进度是否有更新；\
（2）如果有更新的话，记录到项目面板里面，该面板不支持在线更新、仅支持从外部导入；\
（3）项目进度的记录，也和yuque中的逻辑一致，即每个月一条罗列；如果当月有更新、就更新当月的记录；如果进入下一个月，就新增一条下一个月的进度记录；中间的识别逻辑，希望用LLM模型来实现，以增加容错。

***

## 二、数据结构

项目对象新增 `monthlyProgress` 字段：

```json
{
  "monthlyProgress": [
    { "month": "2026年3月", "content": "完成技术方案评审，等待客户确认" },
    { "month": "2026年4月", "content": "合同进入审批流程" }
  ]
}
```

**字段说明：**

- `month`：统一规范化为 `YYYY年M月` 格式（如 `2026年3月`），无论语雀列名写法如何
- `content`：进度文字内容，直接取自语雀表格单元格，不做摘要处理
- 数组按月份**升序**存储，展示时倒序显示（最新在前）

***

## 三、识别流程

### 第一步：列名识别（LLM，调用一次）

把语雀表格的所有列名发给 LLM，识别哪些是进度列并输出标准化月份。LLM 列名识别提前到 `fetchYuqueDoc` 阶段，结果缓存在 `window.yuqueProgressColMap` 里，`confirmYuqueImport` 直接复用。用户在预览阶段就能看到进度信息，进度内容超过 3 行时折叠，显示"展开"按钮。

**Prompt 设计：**

```
以下是一个项目管理表格的列名列表：
{列名JSON数组}

请识别其中属于"项目进度/项目动态/本月进展"类型的列，并将其对应的月份标准化为"YYYY年M月"格式。
规则：
- 如果列名中有完整年月（如"2026年3月"、"2025.6"、"25年3月"），直接提取
- 如果列名中只有月份没有年份（如"3月进展"、"三月进度"），用当前年份 {当前年份} 补全
- 如果无法判断是进度列，返回 null
- 只返回 JSON 对象，key 是原始列名，value 是标准化月份字符串或 null

示例输出：
{
  "本月进展（2026年3月）": "2026年3月",
  "本月进展（2025年6月）": "2025年6月",
  "3月进度": "2026年3月",
  "2025.2进展": "2025年2月",
  "负责人": null,
  "项目名称": null
}
```

**说明：**

- 这一步只调用一次 LLM，不随项目数量增加而增加调用次数
- 结果缓存在内存中，供后续所有项目行复用

### 第二步：提取进度内容

根据第一步识别出的列索引，逐行提取每个项目的进度内容：

- 单元格为空 → 跳过，不生成该月记录
- 单元格有内容 → 生成 `{ month, content }` 记录

### 第三步：合并到本地数据

对每个导入的项目，将语雀进度和本地已有进度合并：

```
对于语雀中每个项目的每条月份进度：
  ├── 本地没有该月记录 → 新增
  ├── 本地已有该月记录 且内容不同 → 覆盖（语雀内容为准）
  └── 本地已有该月记录 且内容相同 → 跳过
```

合并后按月份升序重新排列。

## 四、触发时机

**在语雀导入和Excel 导入时触发**。

触发位置：`confirmYuqueImport` 函数中，与其他字段更新并列处理。

流程：

```
1. 语雀表格列名 → LLM识别进度列（一次调用）
2. 逐项目提取进度内容
3. 与本地 monthlyProgress 合并
4. 保存
```

***

## 五、更新检测与提示

导入完成后，统计本次有进度更新的项目数量，在 toast 里提示：

```
语雀导入：新增 3 个，更新 12 个，其中 5 个项目有新进度
```

## 六、展示设计

### 新增 Tab

在项目编辑面板新增"📋 项目进度" Tab，Tab 顺序根据项目阶段不同：

**洽谈中：**

```
基本信息 / 交付情况 / 项目进度 / 本地文件 / 更新日志
```

**已签单·执行中 / 已完结：**

```
基本信息 / 回款管理 / 交付情况 / 项目进度 / 本地文件 / 更新日志
```

### 展示样式

按月份**倒序**展示，最新进度在最前：

```
┌─────────────────────────────────────┐
│ 📋 项目进度              从语雀同步  │
├─────────────────────────────────────┤
│ 2026年4月                           │
│ 合同进入审批流程，预计5月签订        │
├─────────────────────────────────────┤
│ 2026年3月                           │
│ 完成技术方案评审，等待客户确认       │
│ [展开 ↓]                            │  ← 超过3行时显示
├─────────────────────────────────────┤
│ 2025年12月                          │
│ 完成初步方案，提交报价               │
└─────────────────────────────────────┘
```

- 只读，不支持手动编辑
- 无进度记录时显示"暂无进度记录，请从语雀导入"
- 每条进度内容超过 3 行时折叠，显示"展开 ↓"按钮，点击后展开并显示"收起 ↑"

## 七、LLM 调用时机

列名识别提前到 **`fetchYuqueDoc`** 阶段（读取语雀文档时）执行，而不是等到确认导入时：

```
fetchYuqueDoc()
  ├── 读取文档内容
  ├── 解析表格列名
  ├── 调用 LLM 识别进度列 → 结果存入 window.yuqueProgressColMap
  └── 渲染预览（预览阶段即可看到进度信息）

confirmYuqueImport()
  └── 直接复用 window.yuqueProgressColMap，无需再调用 LLM
```

**好处：**

- 用户在预览阶段就能看到进度识别结果
- 确认导入时无额外等待
